# CLAUDE.md

Этот файл содержит инструкции для Claude Code (claude.ai/code) при работе с кодом в этом репозитории.

## Описание проекта

Telegram-бот для автоматизации заказа еды в Новогрудке, Беларусь. Бот управляется исключительно кнопками (без ввода текста) и использует Playwright для самостоятельного оформления заказов на сайтах доставки.

**Целевой пользователь:** пожилой человек — все решения по UX должны быть максимально простыми, управление только кнопками.

## Стек

- **Python** + **aiogram 3.x** (асинхронный фреймворк для Telegram-ботов)
- **Playwright** (`async_api`) для автоматизации браузера и оформления заказов
- **BeautifulSoup4 + aiohttp** для парсинга меню
- **FSMContext** для управления состоянием диалога
- **Railway.app** (Hobby план) для деплоя через Docker

## Команды

После реализации проекта:

```bash
# Установка зависимостей
pip install -r requirements.txt

# Установка браузеров Playwright
playwright install chromium --with-deps

# Запуск бота
python -m main

# Настройка окружения
cp .env.example .env
# Заполнить .env токеном бота и данными пользователя
```

## Архитектура

### FSM-состояния
Три основных состояния: `restaurant` → `category` → `cart`

### Сценарий использования
```
/start
  → ReplyKeyboard: [«Вкусно вместе», «Сады Победы»]
    → Парсим категории с сайта → InlineKeyboard с категориями
      → Парсим блюда (название + цена) → InlineKeyboard (callback: "add|НАЗВАНИЕ|ЦЕНА")
        → Управление корзиной: [+1, -1, Корзина, Очистить, Оформить заказ]
          → Playwright: открыть сайт → добавить блюда → заполнить форму → отправить → скриншот
```

### Автоматизация заказа (Playwright)
Для каждой позиции в корзине: `page.get_by_text(dish_name)` → клик «Добавить»/«В корзину».
Затем переход в корзину → заполнение фиксированных данных → клик «Оформить заказ» → скриншот подтверждения.

Фиксированные данные заказа хранятся в `.env`:
- Телефон: `+375XXXXXXXXX`
- Адрес: `г. Новогрудок, ул. ______, дом __, кв. __`
- Комментарий: `Перезвонить для уточнения заказа`
- Оплата: `Наличными курьеру`

## Принципы стабильности

- Использовать `page.wait_for_timeout(2000)` между действиями Playwright
- Оборачивать всю автоматизацию в `try/except` с логированием
- Использовать `get_by_role("button", name="Добавить")` как запасной способ поиска элементов
- Логировать все действия бота для отладки
